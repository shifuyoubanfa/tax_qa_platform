①

朴素 / 通用 RAG 基线——它【已经具备】的能力

一套成熟的"单一文档问答"系统该有的部分，当前项目同样保留

基线 = 围绕"文档知识库"的检索问答：把问题理解成意图、按意图混合召回、逐级排序提纯、生成带引用的答案；结构化的精确值用查表 Agent 补。

基线文档混合召回 + 三级排序漏斗

面向非结构化政策/问答文档的检索主链路，这是 RAG 系统的核心，也是基线最扎实的部分。

意图理解 QU→ 混合召回 dense+sparse→ 粗排 RRF 去重→ 精排 bge→ 重排 cross-encoder→ 带 citation 生成
  • ✓ 稠密 + 稀疏混合召回：向量召回 + BM25 全文，按意图配 dense/sparse 权重后归一化融合
  • ✓ 粗排（RRF）：多子查询 / 多库结果倒数排名融合 + 去重
  • ✓ 精排（bge）：按意图选标题 / 正文做语义相似重排序
  • ✓ 重排（cross-encoder）：query×doc 交叉打分定稿 top-k
  • ✓ 带引用生成：依据片段作答、句末标 [citation:x]、附免责声明

基线确定性意图路由 + 结构化查表

用规则优先的意图分类驱动检索策略；社保基数、商品税收分类编码这类"精确值"由查表 Agent 直查数据库并入召回池。

问题→ IntentClassifier 规则优先→ SearchRouter 映射表→ 查哪些库 / 权重
  • ✓ 意图路由：规则/正则优先、LLM 兜底；按意图决定查哪些库与稀疏/稠密权重
  • ✓ 社保结构化查表：ShebaoAgent 直查社保缴费表，命中即作召回结果（结构化优先·跳过文档召回）
  • ✓ 商品编码结构化查表：ProductCodeAgent 直查税收分类编码表，文档 RAG 给不出的精确值由它补
  • ✓ 多库联合召回：政策 / 问答 / 文档 / 案例库可按意图组合召回

说明：查表 Agent 是"召回的一路数据源"，不是平级子智能体——命中即作召回结果（结构化优先，跳过文档召回），统一进排序漏斗；无命中才回退文档 RAG。

②

相对当前项目，基线【尚未具备】的扩展点

★ 这些是把"单一文档问答"撑成"多能力平台"的关键加法

基线的盲区集中在三类：① 结构化数仓的"自然语言查数"；② 多模态文档与对外工具化；③ 联网兜底与流式编排体验。以下是相对当前项目缺的部分。

当前项目独有查数 & 工具化能力

  • ✗ Text2SQL / 数仓自然语言查询：基线只会"在文档里找答案"，问不了"各省本月销售额"这类需要 SQL 聚合的数仓问题
  • ✗ 多模态文档库：基线偏纯文本；当前项目有文档 / 操作手册库槽位（解析含图片切分、图片入对象存储）
  • ✗ MCP 工具化对外：基线只能"自己用"；当前项目把 税务问答 / 社保查询 / 编码查询 / 经营数据查询 暴露成 MCP 工具供外部调用

当前项目独有兜底 & 编排体验

  • ✗ 可答性门控：基线把漏斗挑出的 top-k 无条件喂给 LLM，库里没相关内容时照样"看图说话"硬凑带文号/数字的幻觉；当前项目在重排后加一道确定性阈值门控——重排 top1 相关性分过低/空召回就不喂弱证据、直接诚实兜底（不调 LLM）
  • ✗ 联网补充：基线召回不到就只能说"资料不足"；当前项目有一条默认关、物理隔离的联网旁路兜底（结果绝不进权威引用）
  • ✗ SSE 双流编排：基线常是"算完再返回"；当前项目 updates（意图/检索/可答性/引用/SQL 事件）+ messages（逐 token 答案）双流推送
  • ✗ LangGraph 主图 + 可替换子图：当前项目用图编排，经营数据查询时整条 RAG 被 Text2SQL 子图替换、含 SQL 校正自环

④

扩展性叙事：从"单一文档问答"扩成"多能力平台"

融合两套能力 + 加 MCP / 联网，是设计层的能力扩展，不是性能升级

当前项目并非"把基线调快了"，而是在同一套契约/编排框架下，融合了两套不同形态的能力，再补上对外工具化与联网兜底。

① 多模态文档 RAG 能力

借鉴形态：掌柜智库（多模态文档 RAG）

面向政策/手册/图文文档的检索问答：文档解析（含图片）→切分→稠密+稀疏混合召回→三级排序→带引用生成。这一支正是与基线重合的共同基座，当前项目把它做实并保留。

② Text2SQL / 数仓查询能力

借鉴形态：掌柜问数（LangGraph Text2SQL 数仓查询）

面向结构化数仓的自然语言查数：schema 链接→生成只读 SELECT→AST 安全校验/报错回灌自环修正→执行→数据解读。这一支是基线完全没有的能力，作为可替换子图接入。

两套能力 ＋ 对外/兜底加法 ⟶ 同一平台
★ MCP 工具化对外
把问答 / 社保 / 编码 / 经营数据查询暴露为 MCP 工具，平台从"自己用"变成"能力可被外部编排调用"。
★ 联网补充兜底
默认关、物理隔离、确定性触发的联网旁路，本地召回不足时补来源；结果"未核验"、绝不进权威引用。
★ SSE 双流 + 图编排
LangGraph 主图 + 可替换 Text2SQL 子图，updates / messages 双流推送，边算边吐、进度与引用即时可见。

融合结果：一句话

当前项目在基线"文档混合召回 + 排序漏斗 + 意图路由 + 结构化查表"的共同基座之上，融合掌柜智库（多模态文档 RAG）与掌柜问数（Text2SQL 数仓查询）两套能力，再加上 MCP 对外工具化与联网兜底，由 LangGraph 图编排 + SSE 双流串起，从"单一文档问答"扩成覆盖非结构化检索 + 结构化查数 + 工具化对外 + 联网补充的多能力问答平台。

基线：文档问答（召回·排序·路由·查表） ＋ ★ Text2SQL 数仓查询 ＋ ★ 多模态文档库 ＋ ★ MCP 对外 ＋ ★ 联网兜底 ＋ ★ SSE 双流编排 ＝ 多能力问答平台
